# CoHitch Scaling v0

## Principle
CoHitch should scale by task class and blast radius, not be maximized universally.

## Suggested levels
- Light hitch:
  - ideation
  - copy
  - public render shaping
- Medium hitch:
  - structured docs
  - prompt libraries
  - CoPath creation
- Heavy hitch:
  - policy
  - canon mutation
  - authority / identity / automation escalation

## Rule
Too much hitch for low-risk work creates drag and stall.
Too little hitch for high-risk work creates drift and harm.
