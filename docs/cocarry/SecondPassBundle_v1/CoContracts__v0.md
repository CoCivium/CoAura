# CoContracts v0

## Purpose
Operational physics layer. Sessions are disposable; contracts are durable.

## CoBoard Contract
- local-first by default
- fading unless explicitly kept
- promotion required for persistence or outward routing
- no silent global visibility
- bounded item count required in future implementations

## CoTwilight Contract
- metric is composite, not singular
- if Twilight: recommend CoEx + rotate
- if Critical: force closeout / pointer / death-ready behavior
- telemetry without action is insufficient

## CoRoute Contract
Every routed item should carry:
- source
- destination
- permission scope
- timestamp
- fail / retry expectation

## CoEvo Contract
- mutation unit must be discrete
- artifact must be produced
- artifact must be verifiable
- artifact should be externalizable
- no in-session-only evolution as system norm

## CoPolicy Contract
Single spine for:
- exposure gate
- evolution gate

## Rule
Meta changes require explicit CoEvo, not silent CoEx~.
