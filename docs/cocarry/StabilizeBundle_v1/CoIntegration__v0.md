# CoIntegration v0

## Purpose
Bind CoContracts, CoIdentity, CoBoard, CoRoute, and CoTwilight into one operational model.

## Flow
CoRant → CoBoard → (Keep/Route) → CoRoute → CoObject → CoContracts → CoEx → CoTwilight monitoring

## Identity attachment (optional)
Each CoObject may include:
- CoIdentityPresence
- CoLineageRef
- CoRepTagState

## Rule
No silent evolution. All meaningful changes must be externalized.

## Principle
Sessions are staging. External artifacts are truth.
